<a name="крок-0"></a>
## Крок 0. Що потрібно мати

- **Комп'ютер** (Mac, Windows або Linux).
- **Python 3.10 або новіший** (працює і на 3.14).
  Перевірити: відкрий термінал і набери `python3 --version`. Якщо версія є — добре. Якщо ні — постав із [python.org](https://www.python.org/downloads/).
- **Instagram-акаунт Professional** (Business або Creator). Особистий не підійде.
  Перемкнути: Instagram → Налаштування → Тип акаунта → Переключити на професійний.

> ffmpeg, Homebrew чи інші складні штуки **не потрібні**.

<a name="крок-1"></a>
## Крок 1. Завантажити інструмент


## Крок 2. Встановити залежності

<a name="крок-3"></a>
## Крок 3. Отримати Instagram-токен 🔑

Це найдовший крок, але робиться один раз. Потрібен токен типу **«Instagram API with Instagram Login»** (починається з `IGAA…`).

1. Переконайся, що акаунт **Professional** (див. Крок 0).
2. Зайди на [developers.facebook.com](https://developers.facebook.com) і увійди.
3. **Створи застосунок:** My Apps → Create App → тип **Business** → дай назву.
4. Додай продукт **Instagram** → натисни **API setup with Instagram login**.
5. Знайди розділ **App roles → Roles → Instagram Testers** → **Add people** → впиши свій Instagram-нік → надішли запрошення.
6. **Прийми запрошення** з самого акаунта:
   Instagram → Налаштування → **Apps and websites → Tester invites → Accept**
   (або у браузері: `instagram.com/accounts/manage_access/`).
7. Повернись у **API setup with Instagram login** → навпроти свого акаунта натисни **Generate token**.
8. **Скопіюй** довгий рядок, що починається з `IGAA…`. Це твій токен.

> 💡 **Один застосунок — багато клієнтів.** Щоб додати ще один акаунт (наприклад, клієнта), не створюй новий застосунок — просто додай той акаунт ще одним **Instagram Tester** (крок 5) і згенеруй окремий токен.

> ⏳ Токен діє обмежений час. Коли перестане працювати — згенеруй новий тим самим способом.

<a name="крок-4"></a>
## Крок 4. Налаштувати файл `.env`

1. У теці проєкту зроби копію файлу-прикладу:
   ```bash
   cp .env.example .env
   ```
2. Відкрий `.env` у будь-якому текстовому редакторі.
3. У рядок `INSTAGRAM_TOKEN=` встав свій токен (одразу після `=`, без пробілів і лапок):
   ```
   INSTAGRAM_TOKEN=IGAAxxxxxxxx...
   ```
4. Збережи файл.

Інші рядки — опційні (див. кроки 6-8). Можна поки не чіпати.

<a name="крок-5"></a>
## Крок 5. Запустити дашборд 🚀

```bash
python3 app.py
```

На першому запуску інструмент сам завантажить твої рілси й дописи (це може зайняти хвилину-дві). Коли побачиш рядок `🚀 Дашборд: http://localhost:8080` — відкрий це посилання у браузері.

Готово! Ти побачиш вкладки: **Рілси, Дописи, Огляд, Підписники, Сторіз, Патерни мовлення, Скрипти, Інсайти, Генератор**.

> **Що нового на вкладках:**
> - **Огляд** — порівняння форматів (Рілси/Каруселі/Фото) + теплокарта «найкращий час публікації» + кнопка **«Звіт (PDF)»**.
> - **Підписники** — діапазон дат, оцінка денних **відписок** і **демографія аудиторії** (вік / стать / міста / країни).
> - **Сторіз** — вибір діапазону дат.

> Щоб зупинити — натисни `Ctrl+C` у терміналі. Щоб запустити знову — повтори `python3 app.py`.

<a name="крок-6"></a>
## Крок 6. Генерація скриптів

Вкладка **«Генератор»** пише скрипти у твоєму стилі. Працює **двома способами**:

**Спосіб 1 — без API-ключа (безкоштовно).** Натисни **«📋 Підготувати запит для Claude»** → інструмент збере запит із твоїх топ-рілсів → скопіюй → встав у Claude (застосунок або claude.ai) → скопіюй відповідь назад. Жодного ключа.

**Спосіб 2 — автоматично (з ключем Anthropic).** Якщо хочеш генерацію «однією кнопкою»:
1. Створи ключ на [console.anthropic.com](https://console.anthropic.com) → API Keys (платно, окремо від звичайного claude.ai).
2. Встав його в `.env`: `ANTHROPIC_API_KEY=sk-ant-xxxx...`
3. Перезапусти дашборд.

Без ключа **вся статистика працює** — генератор теж (через Спосіб 1).

<a name="крок-7"></a>
## Крок 7. (Опційно) Автозбір окремих сторіз

Звичайна вкладка «Сторіз» показує **загальну** статистику за період. Якщо хочеш збирати дані по **кожній окремій сторіз** (вони живуть лише 24 год), є скрипт `collect_stories.py`, який треба запускати за розкладом.

**Найпростіше — раз на день вручну:**
```bash
python3 collect_stories.py
```

**Автоматично (macOS, launchd):** створи файл `~/Library/LaunchAgents/com.you.stories.plist`, який двічі на день запускає цей скрипт. Готовий приклад — у розділі «Збирач сторіз» файлу `README.md`.

> Збирає лише **вперед** (історію до старту відновити не можна) і потребує, щоб комп'ютер був увімкнений у момент запуску.

<a name="крок-8"></a>
## Крок 8. Розгортання для кількох клієнтів

Інструмент універсальний — для кожного клієнта це **окрема копія теки** зі **своїм `.env`** (своїм токеном). Єдине, що відрізняє клієнтів, — токен.

Щоб два дашборди працювали одночасно, задай різні порти у їхніх `.env`:
```
# клієнт 1 → .env
PORT=8080
# клієнт 2 → .env
PORT=8081
```

Тоді один дашборд відкриється на `localhost:8080`, інший — на `localhost:8081`.

<a name="крок-9"></a>
## Крок 9. Поділитися з клієнтом (офлайн-файл)

`localhost` працює лише на твоєму комп'ютері — клієнту таке посилання не відкриється. Щоб показати клієнту статистику, можна зібрати **один самодостатній HTML-файл** (`build_export.py`), який відкривається подвійним кліком і працює без сервера.

> Нюанс: мініатюри постів у такому файлі — це посилання на CDN Instagram, які протермінуються за кілька годин/днів. Цифри, графіки й тексти лишаються назавжди.

<a name="проблеми"></a>
## ❓ Що робити, якщо щось не працює

| Симптом | Причина / Рішення |
|---|---|
| `Invalid OAuth access token` | Токен не той тип або протермінувався. Згенеруй новий `IGAA…`-токен (Крок 3). |
| «Недостатня роль розробника» | Акаунт не прийняв запрошення Tester. Прийми його (Крок 3, п. 6). |
| Дашборд показує 0 рілсів / дописів | Перевір, що в `.env` є правильний токен і акаунт — Professional. |
| `command not found: python3` | Python не встановлено. Постав із [python.org](https://www.python.org/downloads/). |
| Порт зайнятий | Інший дашборд уже працює на цьому порту. Задай інший `PORT` у `.env`. |
| Сторіз порожні | У тебе зараз немає активних сторіз або період порожній. Обери інший діапазон дат. |
