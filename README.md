# Instagram Analytics Dashboard

Локальний дашборд аналітики Instagram: підтягує рілси, дописи, сторіз і підписників акаунта через Instagram Graph API, рахує перегляди, охоплення, збереження, репости, ER, візити профілю, нові підписки — і показує це у зручному вебінтерфейсі. Додатково вміє розшифровувати голос у рілсах (faster-whisper) та генерувати скрипти у стилі автора (опційно, через Anthropic Claude).

Працює повністю локально на твоєму комп'ютері. Дані нікуди не надсилаються, окрім офіційного Instagram API.

> 📘 Покрокова інструкція з розгортання — у файлі [`ІНСТРУКЦІЯ.md`](ІНСТРУКЦІЯ.md) (+ її презентаційна PDF-версія в репозиторії).

---

## Можливості

- **Рілси** — ключові метрики, графіки переглядів, таблиця всіх рілсів.
- **Дописи** — аналітика фото й каруселей стрічки: охоплення, перегляди, збереження, **візити профілю** та **нові підписки** з кожного допису.
- **Огляд** — порівняння форматів (Рілси/Каруселі/Фото за середнім охопленням і ER) + теплокарта «найкращий час публікації» (день тижня × година, локальний час) + кнопка **«Звіт (PDF)»**.
- **Підписники** — динаміка приросту за обраний **діапазон дат** (пресети або свій період) + маркери виходу рілсів + оцінка **денних відписок** (кількість) + **демографія аудиторії** (вік, стать, міста, країни).
- **Сторіз** — аналітика за будь-який **діапазон дат** (пресети 7/30/60/90 днів або свій період): охоплення, взаємодії, репости, частка від усього контенту + тренд.
- **Патерни мовлення** — аналіз хуків, слів-магнітів, довжини, розмовності (потрібні транскрипти).
- **Скрипти** — повні розшифровки рілсів.
- **Інсайти** — рекомендації «на що тиснути» на основі твоїх даних.
- **Генератор** — скрипти у твоєму стилі. Працює **без API-ключа** (готує запит → копіюєш у Claude → вставляєш відповідь) або автоматично з ключем Anthropic.

## Вимоги

- **Python 3.10+** (працює і на 3.14).
- Instagram-акаунт **Professional** (Business або Creator).
- Instagram-токен типу **Instagram API with Instagram Login** (див. нижче).
- *(Опційно)* ключ **Anthropic** для ШІ-функцій.

ffmpeg **не потрібен** — розшифровка працює через вбудований у faster-whisper декодер.

## Встановлення

```bash
# 1. Клонувати репозиторій
git clone https://github.com/<your-username>/reels-analytics-dashboard.git
cd reels-analytics-dashboard

# 2. (Рекомендовано) віртуальне середовище
python3 -m venv .venv
source .venv/bin/activate      # macOS / Linux
# .venv\Scripts\activate       # Windows

# 3. Залежності
pip install -r requirements.txt

# 4. Налаштування
cp .env.example .env
# відкрий .env і впиши INSTAGRAM_TOKEN (див. нижче)

# 5. Запуск
python3 app.py
# відкрий у браузері http://localhost:8080
```

На першому запуску дашборд сам підтягне список рілсів.

## Як отримати Instagram-токен

Потрібен токен типу **Instagram API with Instagram Login** (починається з `IGAA…`).

1. Переконайся, що акаунт **Professional** (Instagram → Налаштування → Тип акаунта).
2. Зайди на [developers.facebook.com](https://developers.facebook.com) → створи застосунок (тип **Business**).
3. Додай продукт **Instagram** → **API setup with Instagram login**.
4. У **App roles → Roles → Instagram Testers** додай свій Instagram-нік і надішли запрошення.
5. Прийми запрошення: Instagram → Налаштування → **Apps and websites → Tester invites → Accept**
   (або на вебі: `instagram.com/accounts/manage_access/`).
6. Повернись у **API setup with Instagram login** → **Generate token** навпроти акаунта.
7. Скопіюй токен у `.env` у рядок `INSTAGRAM_TOKEN=`.

> Токен діє обмежений час. Коли протермінується — згенеруй новий тим самим способом.

## Розшифровка голосу (опційно)

У вебінтерфейсі натисни синхронізацію з розшифровкою (whisper). Перший запуск завантажить модель (~140 МБ для `base`). Розмір моделі та мову можна змінити в `.env` (`WHISPER_MODEL`, `WHISPER_LANGUAGE`). Розшифровка вмикає вкладки «Патерни мовлення», «Скрипти» та якісний «Генератор».

---

## ШІ-генерація (опційно)

Впиши `ANTHROPIC_API_KEY` у `.env`. Ключ створюється на [console.anthropic.com](https://console.anthropic.com) (платний, окремо від безкоштовного claude.ai). Без ключа вся статистика працює — вимикається лише вкладка «Генератор».

## Збирач окремих сторіз (опційно)

Instagram віддає дані по **кожній окремій сторіз** лише поки вона активна (24 год). Щоб накопичувати історію по окремих сторіз (а не лише агрегат за 90 днів), є окремий скрипт `collect_stories.py` — він ловить активні сторіз та їхні інсайти й дописує у `stories_media.json`. Запускати треба за розкладом, поки сторіз ще живі (рекомендовано 2×/день).

**Розклад на macOS (launchd):** створи `~/Library/LaunchAgents/com.<you>.reels.stories.plist`, який двічі на день запускає `python3 collect_stories.py` (приклад — у репозиторії історії). Завантаж: `launchctl load -w <plist>`.

> Збирає лише вперед — історію «до старту» відновити не можна. І потребує, щоб комп'ютер був увімкнений у момент запуску (інакше той день пропускається). Для безперебійного збору 24/7 потрібен завжди-увімкнений сервер.

Зібрані сторіз показуються в таблиці «Окремі сторіз» на вкладці «Сторіз».

## Оцінка відписок (опційно)

Instagram **не віддає список підписників**, тож «хто саме відписався» отримати неможливо. Але **кількість** відписок за день можна оцінити: скрипт `collect_followers.py` щодня фіксує загальну кількість підписників у `followers_snapshots.json`, а дашборд рахує `відписки ≈ нові підписники − чиста зміна` і показує це у вкладці «Підписники» (графік «Приріст vs відписки»).

Зручно запускати разом зі збирачем сторіз через обгортку `collect_all.py` (один запуск — обидва збирачі). Працює лише вперед: графік зʼявляється, коли назбирається ≥2 щоденних знімки.
